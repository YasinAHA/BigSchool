# Reglas de Trabajo para Ejercicios de Testing Library & User Events

Este proyecto contiene una carpeta `tasks/` con ejercicios que debes completar **en orden**.
Antes de comenzar, lee completamente este archivo.

---

## 1. Orden de Ejecución

1. Lee `rules.md` (este archivo).
2. Completa los ejercicios dentro de `tasks/` en este orden:
   - Ejercicio 1: Query por Role
   - Ejercicio 2: User Event Simulation

---

## 2. Flujo de Trabajo

### Paso 1 — Leer el ejercicio  
### Paso 2 — Crear Componente  
### Paso 3 — Crear Test  
### Paso 4 — Validación:  
```
npm test
```

---

## 3. Reglas de Estilo

- Prioriza accesibilidad: `getByRole`, `getByLabelText`.
- Evita `getByTestId`.
- Simula interacciones reales con `userEvent`.

---

## 4. Commits

Un commit por ejercicio completado, después de que los tests pasen.
